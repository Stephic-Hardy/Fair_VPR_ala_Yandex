# DemoProject

## 1. Библиотека (де)сереализации JSON формата 
В проекте используется поплуряное решение 'nlohmann': https://github.com/nlohmann/json

Для его установки надо скачать выполнить команду ```sudo apt install nlohmann-json3-dev```. 

## 2. Сборка проекта 

Если собираете проект локально и впервые нужно прописать в командной строке ```git submodule update --init --recursive```

Далее нужно выполнить следующие шаги:
1. ```mkdir build && cd build```
2. ```cmake ..```
3. ```cmake --build .```

Если все прошло успешно - в папке `build/` появится исполняемый файл **app**.

Для сборки в режиме отладки в папке `build/` нужно добавить такой флаг: 
```cmake -DCMAKE_BUILD_TYPE=Debug ..```
Теперь вам будет доступен отладочный вывод.

## 3. Аргументы программы 
Необходимо выполнить следующее: 
```./app -p <problem> -s <solution> -t <time>```
Где:
1. @problem - файл с разрешением `.json`, в котором данные параметры системы.
2. @solution - файл с разерешние `.json`, в который после работы алгоритма будет записано решение.
3. @time - время работы второй части алгоритма в секундах.

Пример:
```./app -p ../tests/data/problem.json -s ../tests/data/mysolution.json -t 10```

## 4. Яндексовские датасеты
В Папке `vrp_problems` лежат 26 файлов в формате json. Их можно использовать при запуске.

## 5. Визуализация работы алгоритма
Написан скрипт `SPB_calculation.py`, который генерит точки внутри Калининского района Санкт-Петербурга, как нормальное распредление с центром в ст. Метро "Политехническая", что более менее соответсвует реальности.

Для запуска необходимо скомпилировать проект по алгоритму выше, а затем из корня репозитория запустить скрипт командой ```python3 SPB_calculation.py```. 

Итогом работы алгоритма будет файл `map_results` (пример которого уже приведён в репозитории). Если его запустить в браузере, то в ресурсе OpenStreetMap будет нарисованы маршруты агентов.
